Password store

What's it for?

A hosted application that lets you create boxes and sub-boxes where you can safely keep passwords. These boxes can be shared among users. This allows you to for instance share a set of passwords with your colleagues.

What features does it have? List them and describe them

Sharing

This is the main goal of this project. A password box can be shared with another user or group of users. This requires user and group management and different levels of users (a group admin for instance).

Backend

This is the server where all passwords are stored. The backend should not be able to decode the stored passwords to minimize trust issues.

Frontend

The parts where you can access and store passwords. These will do some or all of the following items.

  • List boxes and passwords
  • Store and retrieve passwords
  • Manage groups
  • Share boxes

These are some frontend ideas:

  • Website
  • Mobile application
  • Browser plugin
  • Desktop application

How will we organize ourselves? What features can be developed in parallel?

Teams:

  • Backend
  • API (module)
  • Website frontend
  • When resources permit, browser and/or mobile frontend.

Automato

What is it?

Automato is a Python finger for the modern geek.

What does that even mean?

In the old times, we had finger, which provided status reports on a articular computer system or person on a network. Let's bring it back from the dead as a new and shiny API.

The goal of Automato is to let you share information (like displaying what files you are currently working on in your project website) and automate stuff (like launching a pull and test suite run in your office computer, by using a website in your phone).

Automato provides an HTTP API to send and receive information, and two simple clients: command line and javascript. It is up to the user to discover ways to use it.

What features does it have? List them and describe them

Each user can create (and own) a number of apps and channels. Each app can listen to or write on a number of channels. Apps can be open or protected by a secret key.

Every time the user does an API call they must provide the app id (and the secret key if the app is protected).

The owner of an app can add users to a protected app, grant the read and/or write access to each channel that the app uses, and group them in user groups.

Users can identify themselves using oauth.

JSON is used to encode the data.

Examples:

  • Using the Javascript API, I make a simple html page that displays what feature I'm working on (at work), and what's my test coverage. I want to keep this information open only to my colleagues at work, so I add them to my app and I give them access to read the working-on and test-coverage channels.
  • My colleagues sometimes have remarks about the files I'm working on. I add a comment field to my website, and I use the javascript API that sends the users' comments to my comments channel. I grant the user group of my work colleagues write access to this channel. Then, using the command line client, I write a small plugin for my text editor that displays the comments in the status line.
  • Sometimes I see a book that I want to check out later. I add a channel for it, and I write a message to it using the command line client every time I see such a book. I add a section to my blog that displays the books I'm interested in using the Javascript API and adds a link to the Amazon search page for this title.
  • Sometimes while on the go I'm browsing the web and I see a file I want to download, but it's too big to download on my measy and limited mobile data contract. I go to my Automato page and I write a message to my to-download channel with the link to the file. At home, I have a script running that I wrote using the Automato Python client that listens to this client and downloads the file for me. A few minutes later, I check my Automato page and I see that my script at home reports me that the link I posted was wrong. I post the link again (I had messed up with the copy/paste in my phone). Later, I check my Automato page and I see that my home computer reports that the file has been downloaded and is waiting for me at home. I'm happy.
  • While talking to my gatekeeper colleagues, they tell me that a very complex case has been finally merged into the stable branch. I am curious whether the changes in this branch will break my code or have merge conflicts with it, but I don't want to leave the conversation. Luckily, I had added an extra feature to my Working On page that allows me to tell my office computer to pull and merge the stable branch into my code and run the unit tests. I do it. A few minutes later, my Working On page reports that the stable branch is merged and all tests still pass. I continue to enjoy the conversation confident that no nasty merge conflicts are awaiting for me at my desk.

Functionalities (they are the same in the HTTP API, command line client and Javascript client

  • Post a message to a channel
  • Read messages from a channel, optionally deleting them as they are read.
  • Create a user group
  • Assign apis to a user group
  • Give read/write permissions to a user group and channel
  • Create channels
  • Create apps
  • Assign channels to apps and give read/write permissions

Extra functionalities

  • Push notification to a website

We can use Parse or Redis to store the data.

How will we organize ourselves? What features can be developed in parallel?

Teams:

  • REST API
  • Python API/command line client
  • Javascript client
  • Examples (implement some of the examples that I mention here, for example)
impressjs generator from markdown

What is it?

Long time ago, I was working in a generator of slides for impress.js. As a proof of concept the project is working, you can generate slides from markdown, but you can not apply more effect that the default one.

What do I want?

What I want is to finish this project adding support to all the fancy effect that impressjs support. One of the options could be use ReStructured Text instead markdown, and use the extensions to it.

How can we split the work?

For each effect (and they are several, each group should work on them (copy pasted from the tutorial at the end of this document):

data-x
Moves your slide in x-axis left->right or right->left direction depending on the current position. Example: If current x value is 100, data-x='200' will move right->left and data-x='-200' will move left->right.
data-y
Moves your slide in y-axis top->bottom or bottom->top direction depending on the current position. Example: If current y value is 100, data-y='200' will move bottom->top and data-y='-200' will move top->bottom
data-z
Moves your slide in z-axis away from screen or towards the screen depending on the current position. Example: If current z value is 100, data-z='200' will move away from screen and data-z='-200' will move towards the screen.
data-scale
Scales your slides in respect to the other slides. Example: If current scale value is 1, data-scale='4' will scale the slide 4 times bigger than other slides and data-scale='-4' will reduce the scale of the slide to 4 times smaller.
data-rotate-x
Rotates the slide around x axis by given degrees. Example: data-scale-x='90' will rotate the slide 90 degrees clockwise and data-scale-x='-90' will rotate the slide 90 degrees counterclockwise.
data-rotate-y
Rotates the slide around y axis by given degrees. Example: data-scale-y='90' will rotate the slide 90 degrees clockwise and data-scale-y='-90' will rotate the slide 90 degrees counterclockwise.
data-rotate-z
Rotates the slide around z axis by given degrees. Example: data-scale-z='90' will rotate the slide 90 degrees clockwise and data-scale-z='-90' will rotate the slide 90 degrees counterclockwise .data-rotate is equal to data-rotate-z

Extra ball

There are a lot of other JS slides that are really nice like: reveal, that perhaps can be used instead impressjs. This decission is open.

Slides generator

What is it?

Some months ago I saw an application for ipad that make presentations basically letting you do this:

  • Select one image for background in fullscreen.
  • Write one or 2 lines of text that will always fulfill the width of the slide.
  • Change the font family.

What do I want?

A small platform to create this slides and have them always associated to a web with this possibilities:

  • Create an account with Google (or perhaps twitter, facebook...), but Google seems the best option for me now.
  • Create or edit new slides related with this account. This slides creator/editor will have the same functionalities that the ones explained above.
  • Have a permanent link to your slide (completely public for now).

How can we split the work?

  • Create accounts with Google.
  • Editor (IMHO, better to do something with backbone):
    • Font selector.
    • Input to write the content of the slide.
    • JS to always have the font fulfilling all the width.
    • Background selector: for this will be needed to use some public API that allows the user to search for images.
    • JS to always have the background filling all the slide.
  • Saving the slides, perhaps some json format?

My concerns

Perhaps it's a lot of work for only two days hacking, but we can remove something like the font selector, or instead use an API to show the images, use 3 or 4 that we already have (as a proof of concept).

Notes

Sleeper Cell (spy game)

Game for group of people that spend some time with each other daily. Suitable for group working in together in an office. Game Cell consists of 2 Teams (KGB, CIA), people in one Game Cell sould know each other and see each other daily.   • Contact member of other Game Cell (other office in your town).
